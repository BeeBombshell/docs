Rowy uses GCP [service account](https://cloud.google.com/iam/docs/service-account-overview) to perform actions on behalf of you. You can create a service account and set up the project in Rowy with the service account key.

## Steps

1. If you don't have a Firebase/GCP project, create a new project in the [Firebase console](https://console.firebase.google.com/).

2. In [GCP console service account page](https://console.cloud.google.com/iam-admin/serviceaccounts), create a new service account, make sure you are in the correct project and set the service account ID to exactly `rowy-service`.
<img src={imageCreate} alt="Make sure to set service account ID to rowy-service" />

3. In [GCP console IAM page](https://console.cloud.google.com/iam-admin/iam), click grant access button.
<img src={imageGrantAccessButton} alt="Click grant access button" />

4. Then, in the "new principals" input, search for `rowy-service`, select the `rowy-service@<your-project-id>.iam.gserviceaccount.com`.
<img src={imageSearchAccountEmail} alt="Search for rowy-service" />

5. Then, assign the required roles in the "assign roles" input and save. Required roles are:
- App Engine Admin
- App Engine Creator
- Cloud Datastore User
- Cloud Functions Developer
- Cloud Run Admin
- Cloud Scheduler Admin
- Cloud Storage for Firebase Admin
- Eventarc Admin
- Firebase Admin
- Firebase Authentication Admin
- Firebase Rules Admin
- Project IAM Admin
- Pub/Sub Admin
- Secret Manager Admin
- Service Account Admin
- Service Account Token Creator
- Service Account User
- Service Directory Admin
- Service Usage Admin
- Storage Admin
- Storage Object Admin

6. Your service account is ready. It should look like this:
<img src={imageResult} alt="should look like this" />

7. In [GCP console service account page](https://console.cloud.google.com/iam-admin/serviceaccounts), click the `rowy-service` account, in the "keys" tab, click "Add Key" and select "Create new key", select "JSON" type and click "Create".
<img src={imageAddKey} alt="should look like this" />

8. The service account key file should be auto downloaded and is ready for use. It looks like this:
```json
{
  "type": "service_account",
  "project_id": "...",
  "private_key_id": "...",
  "private_key": "-----BEGIN PRIVATE KEY-----...-----END PRIVATE KEY-----\n",
  "client_email": "rowy-service@[your-project-id].iam.gserviceaccount.com",
  "client_id": "...",
  ...
}
```
