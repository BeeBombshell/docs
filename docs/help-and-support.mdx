## Getting Started

If you're new to Rowy, we recommend that you start with the following resources:

- [Rowy Demo Apps](https://demo.rowy.io/): If you're looking for **inspiration**, you can check out our demo apps. These apps are built using Rowy and are available for you to **clone and use as a starting point** for your own projects.

- [Rowy Documentation](https://docs.rowy.io/): We constantly try to improve our documentation to make it more comprehensive. Rowy Docs covers **everything from installation to usage and advanced features**. The documentation also includes code samples, tutorials, and frequently asked questions to help you get started.

- [Rowy GitHub Repository](https://github.com/rowyio/rowy): The official Rowy repository on GitHub, where you can find the **latest release, report issues, and contribute to the project**. The repository will help you stay up-to-date with the latest developments.

- [Rowy Discord Server](https://discord.com/invite/B8yAD5PDX4): Our Discord server is a great place to chat with other Rowy users and get help from the Rowy team. You can join the server by following the link above.


## Troubleshooting

If you're encountering issues with Rowy, the first thing to do is to try to diagnose the problem. Here are some resources that can help:

- [Rowy Community Discussions](https://github.com/rowyio/rowy/discussions): Most of the time, the solution our users are looking for is already available in the community! You can browse existing discussions which seem relevant, or start a new one to get help from the community and the Rowy Team.

- [Rowy GitHub Issue Tracker](https://github.com/rowyio/rowy/issues): If you've encountered a bug or other technical issue with Rowy, you can file an issue on our GitHub repository. This will allow our development team to investigate the issue and provide a fix.

:::info Submitting Issues on GitHub
Before submitting an issue on GitHub, please make sure the issue is not a duplicate. To speed up the resolution process, we appreciate our users and contributors to provide all necessary details in the issue template.
:::

- [Email Support](support@rowy.io): If you'd prefer to contact us via email, you can reach out to our support team at support@rowy.io. We'll do our best to respond to your inquiry as soon as possible.

:::tip Need Help with Cloud Functions?
Most of our users are able to debug their Cloud Functions using the following resources:
- **Using the Rowy Cloud Logs**: You can add `logging.log()` statements to your Cloud Functions to help debug your function logic. [Read more.](../cloud-logs)

- **Checking GCP Cloud Logs**: You can check the [GCP Logs](https://console.cloud.google.com/logs) for your function to check for any specific errors that may be causing your Cloud Function to fail.
:::
